# dynrunner-slurm-shutdown

Standalone, musl-static, single-purpose binary that tears down the
rootless-podman container of a SLURM job after the job ends. Runs on
the compute-node host inside a systemd-user-scope cgroup that survives
SLURM proctrack reap, so it can complete teardown even after `srun` is
gone.

## Why a separate subproject

This binary is fixed infrastructure. It depends on no framework types
and should not be rebuilt every time the framework's Cargo workspace
changes. Living in its own flake means:

- Independent build cache (framework rebuilds do not invalidate it).
- Decoupled distribution: consumers pin a specific shutdown-manager
  revision, separate from their framework version.
- The static musl binary is portable across nix-store rebuilds of
  framework deps.

Accordingly, `shutdown-manager/` is intentionally listed under
`[workspace].exclude` in the repo-root `Cargo.toml`.

## CLI contract

```
dynrunner-slurm-shutdown \
    --container-name <NAME> \
    --storage-root <PATH> \
    --runroot <PATH> \
    --tmp-prefix <PATH> \
    --pid-file <PATH> \
    [--poll-interval-secs 2] \
    [--idle-shutdown-secs 5] \
    [--secondary-grace-secs 5] \
    [--container-stop-grace-secs 10] \
    [--wrapper-pid <PID>] \
    [--log-file <PATH>] \
    [--podman-path <PATH>] \
    [--rm-path <PATH>]
```

The five `--container-name`/paths are required; everything else
defaults. Numeric flags must be > 0 if provided. Both `--flag value`
and `--flag=value` are accepted.

- `--wrapper-pid` ŌĆö when set, the poll loop treats the wrapper
  script's disappearance as a third shutdown trigger (see state
  machine). Omitted ŌćÆ wrapper-monitor disabled.
- `--log-file` ŌĆö append-mode log destination; stderr is always
  written too. Omitted ŌćÆ stderr only.
- `--podman-path` / `--rm-path` ŌĆö absolute binary paths. Omitted ŌćÆ
  literal `podman` / `rm` (PATH lookup).

The wrapper script generated by `crates/dynrunner-slurm` is the
canonical caller; this CLI shape is the contract between the two
subprojects.

## State machine

```
main loop:
  if shutdown flag set      ŌåÆ SIGNAL_SHUTDOWN
  if wrapper_pid gone       ŌåÆ SIGNAL_SHUTDOWN   (only when --wrapper-pid set)
  if container_exists       ŌåÆ saw=true; down_count=0
  else if saw               ŌåÆ down_count++; if down_count Ōēź ceil(idle/poll): IDLE_SHUTDOWN
  sleep(poll_interval); repeat

SIGNAL_SHUTDOWN:
  if container_exists:
    pid = pgrep -P 1 -o          # oldest child of pid 1 inside the container
    if Some(pid): exec kill -TERM pid    (best-effort)
    podman kill --signal TERM <name>     (belt-and-suspenders)
    poll exists once/sec up to secondary_grace
    if still alive: podman stop -t container_stop_grace
  podman rm -af

IDLE_SHUTDOWN:
  podman rm -af

FINAL_CLEANUP (both paths):
  # `podman unshare <rm>` (no --root/--runroot) gated by validate_safe_tmp_path. NO fallback.
  validate_safe_tmp_path(<tmp_prefix>)        # canonicalize + checks (see below)
  podman unshare <rm> <canonical_path> -rf    # uid-0-in-userns bypasses nix-store r-xr-xr-x parents
  unlink <pid_file>                           # missing is OK
```

The `validate_safe_tmp_path` gate is the load-bearing safety
property; it runs BEFORE any exec and fails closed:

1. `canonicalize` resolves symlinks, collapses `..`, and requires
   the path to exist;
2. canonical path is absolute (Unix canonicalize already guarantees
   this; re-asserted defensively);
3. canonical path is valid UTF-8 (`/tmp/asm-*` is ASCII by
   construction);
4. canonical path is strictly under `/tmp/` (refuses bare `/tmp/`
   itself; a symlink whose target leaves `/tmp/` fails here because
   canonicalize already followed it);
5. canonical path does not contain the substring `/home/`;
6. canonical path matches the strict character whitelist
   `[a-zA-Z0-9./_-]` ŌĆö rejects `*`, `'`, `"`, `` ` ``, `$`, `;`,
   `&`, `|`, `<`, `>`, NUL, whitespace, etc.

Argv order is `<rm-path> <canonical-path> -rf` (path BEFORE flags):
if an argv-construction bug ever drops the path slot, `rm -rf`
alone has no operand and is a safe no-op exit-error, rather than
`-rf` surviving into a position that could attach to a subsequent
operand.

SIGNAL_SHUTDOWN has three entry points: the shutdown flag (set by
SIGTERM **or** SIGCONT handlers ŌĆö `signal_hook::low_level::register`,
atomic store, async-signal-safe), or, when `--wrapper-pid` is set,
the wrapper process disappearing (probed with `kill(pid, 0)`). All
three funnel to the same teardown.

Every podman invocation **except** the final-cleanup `unshare`
includes `--root <storage_root> --runroot <runroot>
--cgroup-manager=cgroupfs`. The `unshare` is built without `--root`/
`--runroot` on purpose: passing them initializes a storage driver
that holds a lock the nested `rm` then deadlocks on.

## Module map

| Module | Single concern |
| --- | --- |
| `config` | argv ŌåÆ `Config` |
| `podman` | `PodmanBackend` trait + `RealPodman` |
| `shutdown_flag` | `AtomicBool` flag set by signal handlers |
| `signals` | install SIGTERM/SIGCONT handlers on the flag |
| `clock` | `Clock` trait + `RealClock` for testable sleeps |
| `process_probe` | `ProcessProbe` trait + `KillProbe` (`kill(pid,0)`) ŌĆö wrapper-PID liveness, the loop's third wake input |
| `poll_loop` | state machine |
| `cleanup` | `/tmp` removal + PID-file lifecycle |
| `testing` | `MockBackend` + `FakeClock` + `MockProcessProbe` shared by unit + integration tests (LTO-stripped from the production binary) |

## Building

Canonical build (musl-static, what consumers ship):

```
nix build .#default --print-build-logs
file result/bin/dynrunner-slurm-shutdown   # ŌåÆ "static-pie linked"
ldd result/bin/dynrunner-slurm-shutdown    # ŌåÆ "statically linked"
```

Native development build (glibc-linked, fine for tests):

```
cargo build --release
cargo test
```

Both must pass clean.

### Release-profile settings

`[profile.release]` in `Cargo.toml` is tuned for binary size:

- `opt-level = "z"` ŌĆö optimise for size
- `lto = "fat"` ŌĆö whole-binary LTO; removes the `testing` module from the production output
- `panic = "abort"` ŌĆö no unwinder
- `strip = true` ŌĆö drop symbols
- `codegen-units = 1` ŌĆö better inlining at the cost of build time
- `overflow-checks = false` ŌĆö release default; documented explicitly

The musl-static binary is well under 1 MB stripped.

## Dependencies

Deliberately minimal:

- `signal-hook` v0.3, `default-features = false` ŌĆö single async-signal-safe atomic-flag setter.
- `libc` v0.2, `default-features = false` ŌĆö `kill(pid, 0)` + `__errno_location` for `process_probe::KillProbe`.
- `tempfile` (dev only) ŌĆö temp dirs for filesystem tests.

No tokio. No async runtime. Standard library `Command` for podman
invocations, `thread::sleep` for the poll cadence. Argv parsing is
hand-rolled.

## Testing

`cargo test` runs 66 tests (50 unit, 16 integration) covering:

- PID-file write/remove lifecycle.
- Idle shutdown after grace, *only* following at-least-one container sighting.
- Signal shutdown via both `pgrep -P 1 -o ŌåÆ Some` and `ŌåÆ None` paths.
- Wrapper-monitor: inert when `--wrapper-pid` is unset; triggers SIGNAL_SHUTDOWN the tick the probe reports the PID gone.
- Belt-and-suspenders `podman kill` always fires after the in-container kill.
- Graceful exit within grace skips `podman stop`; surviving the grace triggers it.
- `final_cleanup` runs on both paths via `podman unshare <rm> -rf` (no --root/--runroot) gated by `validate_safe_tmp_path`.
- Config parsing: missing required flags, unknown flags, zero/non-numeric values, `--wrapper-pid` range, optional-flag defaults.

Tests use the in-process `testing::MockBackend`, `testing::FakeClock`,
and `testing::MockProcessProbe`; no real `podman` is spawned and no
real `thread::sleep` runs, so the full suite completes in milliseconds.
